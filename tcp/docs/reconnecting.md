리커넥트의 동작방식

먼저 기반이 되어야 하는건
1. 게임 세션이 현제 나가면 자동 으로 세션이 마무리 되는데 이걸 안되도록 해야하고
2. 클라이언트 상태에 리커넥트 상태라는걸 만들어야한다.

일반적이 경우에는 연결이 끊기면 
바로 Disconnected되고 종료 상태가 되는데

게임 세션에서 연결이 끊길경우 
Reconnecting 과 같은 상태로 클라이언트 상태가 설정됨

그리고 이상태동안 재연결 되면 게임 세션에 재참여가됨
1. clientSession정보 이어받기
2. 게임 세션 정보 전달 받기


만약 게임 세션이 종료될때 까지 리커넥티 상태 였다 
그러면 해당 클라이언트 세션은 디스커넥트 처리됨

(즉 현제 디스커넥트 처리되고 따라서 패배 처리하게 되는 해당 부분을 그대로 진행하게됨 대신 해당 부분에서
재연결 상태 -> 연결끊김 상태 전환 기능이 추가되고)
